# Product Brief: Enometa Shopping Mall

## Executive Summary

Enometa는 감도 높은 프리미엄 컨템포러리 패션 쇼핑몰 포트폴리오다. 단순히 "예쁜 쇼핑몰 화면"을 만드는 것이 아니라, 상품 탐색부터 결제까지 실제로 동작하는 풀스택 이커머스 시스템을 구축한다.

le17septembre.com의 미니멀 럭셔리 감성을 기반으로 하되, **인터랙티브 반응형 경험**을 결합하여 차별화한다. 인트로 시퀀스, 페이지 트랜지션, 호버 모션 강화, 사이드 드로어 장바구니 등으로 "정적인 우아함"을 넘어 "반응하는 우아함"을 구현한다.

이 프로젝트는 두 가지 목적을 동시에 달성한다: 강의 미션("동작하는 쇼핑몰")의 완성도 있는 수행과, 실제 외주 클라이언트에게 보여줄 수 있는 포트폴리오 구축.

## The Problem

쇼핑몰 개발 외주 시장에서 포트폴리오가 없으면 첫 수주를 받기 어렵다. 대부분의 주니어 개발자 포트폴리오는:

- **화면만 있고 동작하지 않는** 목업 수준
- **디자인 감각이 부족한** 기본 부트스트랩 템플릿
- **체계적 과정 없이** 대충 만든 결과물
- **기술적 깊이가 보이지 않는** 단순 CRUD

외주 클라이언트는 3분 안에 "이 사람한테 맡겨도 되겠다"를 판단한다. 그 3분에 기술력, 디자인 감각, 운영 이해도를 동시에 보여줘야 한다.

## The Solution

**Enometa** — 프리미엄 컨템포러리 패션 브랜드를 위한 온라인 쇼핑몰.

핵심 경험:
- **인트로 시퀀스**: 브랜드 세계관에 진입하는 연출 (타이핑 애니메이션 + 이미지 카드 시퀀스)
- **상품 탐색**: 카테고리 필터링, 그리드 전환(2/3/4열), 호버 이미지 스왑 + 미리보기
- **상품 상세**: 에디토리얼 이미지 갤러리, 옵션 선택(사이즈/컬러), Sticky 구매 바
- **구매 플로우**: 사이드 드로어 장바구니 → 토스페이먼츠 실결제 (테스트 모드)
- **고객 소통**: 자체 문의 시스템 (플로팅 버튼 → 폼 → DB 저장 → 관리자 조회)
- **운영 관리**: Notion 스타일 미니멀 관리자 대시보드

## What Makes This Different

**1. 인터랙티브 반응형 경험**
일반 쇼핑몰 포트폴리오는 정적 페이지의 나열이다. Enometa는 Framer Motion 기반 페이지 트랜지션, 호버 모션 강화, 스크롤 애니메이션으로 "앱 같은" 경험을 제공한다.

**2. 실제 동작하는 풀스택**
화면만 있는 목업이 아니라, Supabase(PostgreSQL) 기반 실제 데이터 + 토스페이먼츠 결제 연동 + 자체 문의 시스템까지 프론트-백-관리자가 모두 연결된다.

**3. 체계적 개발 과정 문서화**
BMAD 방법론을 적용한 브레인스토밍 → 브리프 → PRD → 아키텍처 → 스프린트 전체 과정이 문서화되어, "이 사람은 체계적으로 일한다"는 것을 증명한다.

**4. AI 활용 이미지 생성**
Midjourney + Gemini 3 Pro로 에디토리얼 패션 감성의 상품 이미지를 직접 생성. 디자이너 없이도 하이엔드 비주얼을 만들 수 있다는 것을 보여준다.

## Who This Serves

**1차 타겟: 강의 미션 평가자**
- "동작하는 쇼핑몰"을 기대하며, 과정과 열정을 평가
- 쇼핑몰 구조 이해도, 학습 과정, 완성도를 본다

**2차 타겟: 외주 클라이언트 (패션/라이프스타일 브랜드)**
- 자사 브랜드의 온라인 쇼핑몰 구축을 원함
- 디자인 감각 + 기술력 + 운영 이해를 갖춘 개발자를 찾는 중
- 포트폴리오를 통해 3분 안에 "맡길 수 있겠다"를 판단

## Success Criteria

| 기준 | 목표 |
|------|------|
| 풀 플로우 동작 | 상품 탐색 → 장바구니 → 결제 → 주문 완료가 끊김 없이 동작 |
| 페이지 완성도 | 인트로 + 상세 95%, 리스트 90%, 장바구니/결제 85%, 관리자 80% |
| 상품 수 | 12~15개 (카테고리당 2~3개, 고퀄 상세 2개) |
| 디자인 일관성 | 프리미엄 컨템포러리 톤이 전 페이지에서 유지 |
| 반응형 | 데스크톱 + 모바일 모두 정상 동작 |
| 과정 문서 | BMAD 전 단계 산출물 + 최종 과정 보고서 완성 |
| 배포 | Vercel에 라이브 배포 완료, 접속 가능한 URL 제공 |

## Scope

### In (주말 스프린트)
- 인트로 페이지 (타이핑 + 카드 시퀀스 + 입장)
- 상품 리스트 (카테고리 필터, 그리드 전환, 호버 미리보기)
- 상품 상세 (이미지 갤러리, 옵션 선택, Sticky Bar)
- 사이드 드로어 장바구니
- 결제 (토스페이먼츠 테스트 모드)
- 회원가입/로그인 (Supabase Auth)
- 마이페이지 (주문 내역 조회)
- 자체 문의 시스템 (플로팅 버튼 → 폼 → DB)
- 관리자 대시보드 (상품 CRUD, 주문 관리, 문의 조회, 기본 통계)
- AI 생성 상품 이미지 12~15세트
- 과정 문서화 (BMAD 산출물 + 최종 보고서)
- Vercel 배포

### Out (주말 스코프 밖)
- 공지 배너 / 프로모션 슬라이드
- 다국어 지원 (EN/KR)
- 멤버십 등급별 가격 체계
- 재입고 SMS 알림
- 네이버 마일리지 / 외부 포인트 연동
- 시즌별 컬렉션 분류
- 소셜 로그인 (카카오/네이버/구글) — 이메일 로그인만
- 실제 배송 추적 연동
- 리뷰/Q&A 게시판 (시간 남으면 추가)

## Vision

이 프로젝트가 성공하면:

**단기** — 강의 미션을 높은 평가로 완수하고, 즉시 사용 가능한 포트폴리오 확보.

**중기** — 이 코드베이스를 기반으로 실제 외주 수주 시 빠르게 커스터마이징. 브랜드 톤만 바꾸면 다른 패션 쇼핑몰로 전환 가능한 구조.

**장기** — "쇼핑몰 전문 개발자"로서의 포지셔닝. 이 포트폴리오 + BMAD 과정 문서를 통해 체계적 개발 능력을 증명.
